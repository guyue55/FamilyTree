# 族谱页面UI设计稿说明

## 📋 设计概述

本设计稿是基于产品需求文档(PRD)和UI设计文档创建的族谱页面完整可视化原型，展示了现代化的族谱管理系统界面设计。

## 🎯 设计目标

1. **传统与现代结合** - 保持中华传统文化特色，融入现代化UI设计理念
2. **功能完整性** - 涵盖族谱展示、成员管理、关系查询等核心功能
3. **用户体验优先** - 简洁直观的操作界面，降低学习成本
4. **响应式设计** - 适配桌面端、平板和移动端设备
5. **可扩展性** - 模块化设计，便于后续功能扩展

## 🏗️ 页面布局结构

### 1. 顶部导航栏 (Header)
- **品牌标识**: 族谱系统Logo和家族名称
- **面包屑导航**: 清晰的页面层级导航
- **全局搜索**: 支持成员姓名和关键词搜索
- **用户操作**: 通知、设置、用户菜单

### 2. 工具栏 (Toolbar)
- **布局控制**: 垂直树形、水平树形、径向布局、力导向
- **世代筛选**: 支持按世代范围筛选显示
- **成员搜索**: 快速定位特定成员
- **称呼查询**: 一键查询成员间的称呼关系
- **缩放控制**: 放大、缩小、适应屏幕
- **显示选项**: 照片、日期、世代标识的显示控制
- **导出功能**: 支持族谱图形导出

### 3. 侧边栏 (Sidebar)
#### 族谱信息卡片
- 家族基本信息（创建者、创建时间、祖籍地、描述）
- 统计数据（总成员数、世代数、男女比例）

#### 筛选器
- 性别筛选（全部/男性/女性）
- 在世状态筛选（全部/在世/已故）
- 婚姻状态筛选（全部/已婚/未婚）

#### 成员列表
- 成员头像和基本信息
- 世代标识和生卒年份
- 快速操作按钮（编辑、查看）

### 4. 主图形展示区域 (Graph Container)
#### 族谱节点设计
- **节点样式**: 圆角矩形卡片，区分男女性别
- **节点内容**: 头像、姓名、生卒年份、世代标识
- **交互状态**: 悬停、选中、高亮效果

#### 连接线设计
- **血缘关系**: 蓝色实线连接
- **婚姻关系**: 粉色虚线连接
- **层次清晰**: 不同世代间的垂直连接

#### 图形控制
- **缩略图导航**: 右下角小地图，快速定位
- **控制按钮**: 适应屏幕、居中显示、全屏模式

### 5. 底部状态栏 (Status Bar)
- 总成员数、世代数统计
- 当前缩放比例
- 选中成员信息
- 最后更新时间

## 🎨 视觉设计特色

### 色彩系统
- **主色调**: 中国红 (#C8102E) - 体现传统文化
- **辅助色**: 金黄色 (#D4AF37) - 增加尊贵感
- **功能色**: 蓝色、绿色、橙色、红色 - 区分不同状态
- **中性色**: 灰色系 - 保证内容可读性

### 字体系统
- **主字体**: PingFang SC / Microsoft YaHei
- **字重**: 常规(400)、中等(500)、粗体(600)
- **字号**: 10px-20px 层次化设计

### 间距系统
- **基础间距**: 4px、8px、16px、24px、32px
- **统一规范**: 保证视觉一致性

### 圆角和阴影
- **圆角**: 4px、6px、8px 渐进式设计
- **阴影**: 三级阴影系统，增强层次感

## 🔧 技术实现要点

### 前端技术栈
- **框架**: Vue 3.4+ + TypeScript
- **UI组件**: Element Plus 2.4+
- **图形渲染**: G6 (AntV) 4.8+
- **状态管理**: Pinia 2.1+
- **路由**: Vue Router 4.2+

### 关键组件设计
1. **FamilyTreeGraph** - 族谱图形主组件
2. **TreeNode** - 族谱节点组件
3. **MemberSidebar** - 成员侧边栏组件
4. **FilterPanel** - 筛选器面板组件
5. **ToolbarControls** - 工具栏控制组件

### 状态管理
- **familyStore** - 族谱数据状态
- **uiStore** - 界面状态管理
- **userStore** - 用户信息状态
