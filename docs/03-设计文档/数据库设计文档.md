## 1. 设计概述

### 1.1 设计原则
- **逻辑关联**: 不使用物理外键，采用逻辑关联维护数据关系
- **性能优先**: 合理设计索引，优化查询性能
- **扩展性**: 预留扩展字段，支持功能扩展
- **数据完整性**: 应用层保证数据一致性和完整性
- **安全性**: 敏感数据加密存储

### 1.2 命名规范
- **表名**: 使用复数形式，下划线分隔，如 `family_trees`
- **字段名**: 使用下划线分隔，如 `create_time`
- **索引名**: `idx_表名_字段名`，如 `idx_users_email`
- **主键**: 统一使用 `id` 作为主键名

### 1.3 数据类型规范
- **主键**: `int unsigned` 自增
- **外键关联**: `int unsigned`
- **状态字段**: `tinyint(1)` 
- **字符串**: `varchar(n)` 根据实际需要设置长度
- **文本**: `text` 用于长文本内容
- **时间**: `datetime` 格式为 'YYYY-MM-DD HH:MM:SS'
- **金额**: `int` 以分为单位存储

## 2. 数据库表设计

### 2.1 用户管理相关表

#### 2.1.1 用户表 (users)
```sql
CREATE TABLE users (
    id int unsigned NOT NULL AUTO_INCREMENT COMMENT '用户ID',
    username varchar(50) NOT NULL COMMENT '用户名',
    email varchar(100) NOT NULL COMMENT '邮箱',
    phone varchar(20) DEFAULT NULL COMMENT '手机号',
    password_hash varchar(255) NOT NULL COMMENT '密码哈希',
    real_name varchar(50) DEFAULT NULL COMMENT '真实姓名',
    avatar_url varchar(255) DEFAULT NULL COMMENT '头像URL',
    gender tinyint(1) DEFAULT NULL COMMENT '性别：1-男，2-女，0-未知',
    birth_date date DEFAULT NULL COMMENT '出生日期',
    status tinyint(1) NOT NULL DEFAULT 1 COMMENT '状态：1-正常，0-禁用',
    last_login_time datetime DEFAULT NULL COMMENT '最后登录时间',
    create_time datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_users_username (username),
    UNIQUE KEY uk_users_email (email),
    KEY idx_users_phone (phone),
    KEY idx_users_status (status),
    KEY idx_users_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

#### 2.1.2 用户角色表 (user_roles)
```sql
CREATE TABLE user_roles (
    id int unsigned NOT NULL AUTO_INCREMENT COMMENT '角色ID',
    user_id int unsigned NOT NULL COMMENT '用户ID',
    family_tree_id int unsigned NOT NULL COMMENT '族谱ID',
    role_type tinyint(1) NOT NULL COMMENT '角色类型：1-创建者，2-管理员，3-编辑者，4-查看者',
    permissions json DEFAULT NULL COMMENT '权限配置JSON',
    status tinyint(1) NOT NULL DEFAULT 1 COMMENT '状态：1-正常，0-禁用',
    create_time datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_user_roles_user_family (user_id, family_tree_id),
    KEY idx_user_roles_user_id (user_id),
    KEY idx_user_roles_family_tree_id (family_tree_id),
    KEY idx_user_roles_role_type (role_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户角色表';
```

## 3. 索引设计

### 3.1 主键索引
- 所有表都使用 `id` 作为主键，自动创建主键索引

### 3.2 唯一索引
- `users.username`: 用户名唯一
- `users.email`: 邮箱唯一
- `user_roles(user_id, family_tree_id)`: 用户在族谱中的角色唯一
- `kinship_titles(relationship_path, dialect, gender_context)`: 称呼路径唯一

### 3.3 普通索引
- 外键关联字段索引
- 状态字段索引
- 时间字段索引
- 查询频繁的字段索引

### 3.4 复合索引
- `user_roles(user_id, family_tree_id)`: 用户角色查询
- `relationships(family_tree_id, person_a_id)`: 关系查询
- `family_members(family_tree_id, generation)`: 世代查询
