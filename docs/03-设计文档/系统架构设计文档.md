## 1. 架构概述

### 1.1 架构目标
- **高可用性**: 系统可用性达到99%以上
- **高性能**: 支持1000并发用户访问
- **可扩展性**: 支持水平扩展和功能扩展
- **安全性**: 数据安全和用户隐私保护
- **可维护性**: 代码结构清晰，便于维护

### 1.2 架构原则
- **分层架构**: 表现层、业务层、数据层分离
- **模块化设计**: 功能模块独立，低耦合高内聚
- **标准化**: 遵循RESTful API设计规范
- **安全优先**: 所有接口都有权限控制

## 2. 技术选型

### 2.1 后端技术栈
- **编程语言**: Python 3.12
- **Web框架**: Django 4.2 + Django-Ninja
- **数据库**: SQLite (开发) / MySQL (生产)
- **缓存**: Redis
- **任务队列**: Celery
- **Web服务器**: Gunicorn + Nginx

### 2.2 前端技术栈
- **框架**: Vue.js 3
- **UI组件库**: Element Plus
- **构建工具**: Vite
- **状态管理**: Pinia
- **图形库**: G6 (AntV)
- **HTTP客户端**: Axios

### 2.3 开发工具
- **版本控制**: Git
- **代码规范**: ESLint + Prettier (前端), Black + Flake8 (后端)
- **测试框架**: Jest (前端), Pytest (后端)
- **文档工具**: Swagger/OpenAPI

### 3.2 模块架构

#### 3.2.1 用户管理模块
- **用户注册/登录**: 支持多种注册方式
- **权限管理**: 基于角色的权限控制
- **用户信息**: 个人资料管理

#### 3.2.2 族谱管理模块
- **族谱CRUD**: 族谱的增删改查操作
- **成员管理**: 族人信息的录入、编辑和维护
- **关系管理**: 血缘关系和婚姻关系的建立和维护
- **权限管理**: 族谱的分享权限和协作权限控制
- **用户关联**: 族人节点与系统真实用户的关联管理

#### 3.2.3 可视化展示模块
- **树形渲染**: 基于G6的族谱树形结构可视化
- **交互控制**: 缩放、拖拽、搜索、导航等交互功能
- **布局算法**: 多种族谱布局方式（垂直树、水平树、辐射树等）
- **节点展示**: 族人信息的节点展示和详情查看
- **关系展示**: 血缘关系和婚姻关系的可视化连线
- **SVG连线系统**: 实现父子关系和夫妻关系的差异化连线
  - 父子关系：蓝色实线连接
  - 夫妻关系：粉色虚线连接
  - 连线交互：支持hover效果和显示/隐藏切换
- **UI优化**: 世代标签颜色优化、成员卡片名称居中对齐
- **响应式设计**: 支持多设备适配和移动端优化

#### 3.2.4 称呼计算模块
- **关系计算**: 计算任意两个族人之间的关系路径
- **称呼映射**: 关系路径到具体称呼的转换
- **方言支持**: 多方言称呼系统和地区差异支持
- **双向查询**: 支持A对B和B对A的双向称呼查询

## 4. 数据架构

### 4.1 数据库设计原则
- **逻辑关联**: 不使用外键，采用逻辑关联
- **索引优化**: 合理设计索引提高查询性能
- **数据完整性**: 应用层保证数据一致性
- **扩展性**: 预留扩展字段

### 4.2 核心数据表

#### 4.2.1 用户相关表
```sql
-- 用户表
users (id, username, email, phone, password_hash, status, create_time, update_time)

-- 用户角色表
user_roles (id, user_id, role_type, family_tree_id, create_time)
```

#### 4.2.2 族谱相关表
```sql
-- 族谱表
family_trees (id, name, description, creator_id, status, create_time, update_time)

-- 族谱成员表
family_members (id, family_tree_id, name, gender, birth_date, death_date, 
                photo_url, description, status, create_time, update_time)

-- 关系表
relationships (id, family_tree_id, person_a_id, person_b_id, 
               relationship_type, create_time, update_time)
```

### 4.3 缓存策略
- **用户会话**: Redis存储用户登录状态

- **称呼计算**: 计算结果缓存
- **静态资源**: CDN缓存图片等静态文件
