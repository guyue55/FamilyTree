## 1. 项目概述

### 1.1 项目简介
族谱系统是一个专业的家族族谱管理平台，支持多用户、多族谱管理，提供可视化族谱展示和智能称呼计算功能。

### 1.2 技术架构
- **架构模式**: 前后端分离
- **前端**: Vue.js 3 + Element Plus + G6
- **后端**: Django + Django-Ninja
- **数据库**: MySQL + Redis
- **部署**: Docker + Nginx

## 5. API设计规范

- 使用标准的HTTP方法：GET、POST、PUT、DELETE
- URL使用名词复数形式，避免动词
- 使用HTTP状态码表示请求结果
- 统一的响应格式

### 8.2 模型设计规范
```python
from django.db import models
from django.contrib.auth.models import AbstractUser


class BaseModel(models.Model):
    """基础模型类，包含公共字段。"""
    
    created_at = models.DateTimeField(auto_now_add=True, verbose_name='创建时间')
    updated_at = models.DateTimeField(auto_now=True, verbose_name='更新时间')
    
    class Meta:
        abstract = True


class FamilyTree(BaseModel):
    """族谱模型。"""
    
    name = models.CharField(max_length=100, verbose_name='族谱名称')
    description = models.TextField(blank=True, verbose_name='族谱描述')
    # 使用BigIntegerField进行逻辑关联，不使用ForeignKey
    user_id = models.BigIntegerField(
        help_text='创建者ID，逻辑关联users.id',
        verbose_name='创建者ID'
    )
    is_public = models.BooleanField(default=False, verbose_name='是否公开')
    
    class Meta:
        db_table = 'family_trees'
        verbose_name = '族谱'
        verbose_name_plural = '族谱'
        ordering = ['-created_at']
        indexes = [
            models.Index(fields=['user_id']),
        ]
    
    def __str__(self):
        return self.name
    
    @property
    def user(self):
        """获取关联的用户对象"""
        from django.contrib.auth import get_user_model
        User = get_user_model()
        return User.objects.get(id=self.user_id)
```

### 8.3 视图规范
```python
from ninja import Router
from django.shortcuts import get_object_or_404
from .models import FamilyTree
from .schemas import FamilyTreeSchema, FamilyTreeCreateSchema


router = Router()


@router.get("/", response=List[FamilyTreeSchema])
def list_family_trees(request):
    """获取族谱列表。
    
    Returns:
        List[FamilyTreeSchema]: 族谱列表
    """
    trees = FamilyTree.objects.filter(user_id=request.user.id)
    return trees


@router.post("/", response=FamilyTreeSchema)
def create_family_tree(request, data: FamilyTreeCreateSchema):
    """创建族谱。
    
    Args:
        data: 族谱创建数据
        
    Returns:
        FamilyTreeSchema: 创建的族谱信息
    """
    tree = FamilyTree.objects.create(
        user_id=request.user.id,
        **data.dict()
    )
    return tree
```

## 11. 版本控制规范

### 11.1 分支管理
- `main`: 主分支，用于生产环境
- `develop`: 开发分支，用于集成开发
- `feature/*`: 功能分支，用于新功能开发
- `bugfix/*`: 修复分支，用于bug修复
- `hotfix/*`: 热修复分支，用于紧急修复

### 11.2 提交规范

#### 11.2.1 提交格式
Git commit 要清晰明了的说明本次提交的目的。

**格式**: `[type](scope): subject`

**示例**:
- `[feature](family): add quality field`
- `[bugfix](auth): fix user login validation`

#### 11.2.2 提交类型（Type）
[强制] type 用于说明 git commit 的类别，只允许使用下面的标识：

- **[feature]**: 新功能（feature）
- **[bugfix]**: 修复bug，可以是QA发现的BUG，也可以是研发自己发现的BUG
  - [bugfix]: 产生diff并自动修复此问题。适合于一次提交直接修复问题
  - to: 只产生diff不自动修复此问题。适合于多次提交。最终修复问题提交时使用fix
- **[docs]**: 文档（documentation）
- **[format]**: 格式（不影响代码运行的变动）
- **[refactor]**: 重构（即不是新增功能，也不是修改bug的代码变动）
- **[performance]**: 优化相关，比如提升性能、体验
- **[test]**: 增加测试
- **[chore]**: 构建过程或辅助工具的变动
- **[migrate]**: 代码迁移

#### 11.2.3 作用域（Scope）
scope 用于说明 commit 影响的范围，比如数据层、控制层、视图层等等，视项目不同而不同。

#### 11.2.4 主题（Subject）
[强制] subject是commit目的的简短描述，不超过50个字符：
- 优先使用英文，也可以使用中文
- 结尾不需要标点符号

#### 11.2.5 提交方式
- 所有项目的 MR 都强制要求 MR 方式为 rebase & squash，所以每个 MR 提交到默认分支之后，只会有 1 个 commit
- 大家务必一个 MR 只做一件事，一件事只提一个 MR

#### 11.2.6 分支命名规范
- 把分支名定义为 `feature/{模块}/add-user` 的样式
- 把 MR 和 commit 的 title 定义为 `[feature](模块): 功能描述` 的格式

#### 11.2.7 提交示例
```
[feature](family): 添加族谱可视化功能
[bugfix](auth): 修复用户登录验证问题
[performance](api): 优化族谱查询性能
[refactor](models): 重构用户模型结构
[format]: 代码格式化调整
[docs]: 更新API文档
[test]: 添加用户认证单元测试
[chore]: 更新依赖包版本
```

#### 11.2.8 代码规范要求
- 单元测试覆盖率 ≥ 70%

### 11.3 代码审查规范
- 所有代码必须经过Code Review
- 至少需要一个审查者批准
- 检查代码规范、逻辑正确性和测试覆盖

## 12. 部署规范

### 12.1 环境配置
- 开发环境：本地开发和调试
- 测试环境：功能测试和集成测试
- 预生产环境：性能测试和用户验收测试

### 12.2 Docker化部署
```dockerfile
# Dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["gunicorn", "config.wsgi:application", "--bind", "0.0.0.0:8000"]
```

### 12.3 CI/CD流程
1. 代码提交触发自动构建

3. 构建Docker镜像
4. 部署到测试环境
5. 运行集成测试
6. 部署到生产环境
